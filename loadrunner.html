LoadRunner Transaction Adder (Excel Macro)

Some LoadRunner Perf tester has practice to add the transactions at the end after validation. I think, this will be very useful for them. Please download the excel macro below.

LR_TransAdder

About this Macro

  • Unzip and save the folder in C drive and open the Excel macro file. Click "Browse File" button to select your script file (Action.c) and mention your transaction prefix.
  • Click "Load Commands" button to load all your commands from the script, transaction prefix and Thinktime will be added.
  • In start transaction, you can mention your action names after transaction prefix. It will automatically add the same in end transaction.
  • After adding transaction name, you can click "Add Transaction" button. Now your script will be embedded with transaction names.
  • Your original script file (Action.c) copy will be saving in LR_TransAdder\Temp location. If you want to revert back your original script, you can copy paste from here.

Web Service Scripting

One of the way for web services scripting is using web_custom_request() function. You have to format your xml request before passing it to body of custom request function. Please click below to open webservices scripter.

LR_WSSC


web_custom_request("Name",
"URL=endpoint",
"Method=POST",
"TargetFrame=",
"Resource=0",
"RecContentType=text/xml",
"Referer=",
"Mode=HTML",
"EncType=text/xml;Â charset=utf-8",
"Body=XML Request"
LAST);

Changes required in xml request

  • Tab field should be removed ("\t" replace with white space)
  • New line character should be removed ("\n" replace with white space)
  • " should be replaced with \"

You can use textpad and notepad++ to format these changes.

Here you go to format more number of xml in jest a single click

  • Click above button to open "Webservices Scripter"
  • It will open a page where you can copy/paste your xml requests. And it will perform above mentioned changes in single click. (Simple javascript used to format this changes :) )

Truclient Tips

Dynamic Checkbox Selection

If we want to select dynamic number of checkboxes (change based on users or any parameter), we can use the following JavaScript function to check the checkboxes.

  • Find the checkbox table name using inspects element or firebug.
  • Use the "document.getElementById("tablename").rows.length" in for loop to iterate number of rows.
  • Add the click option inside the for loop with option selection "i"

Random Link Selection

If you want select random link or radio button from the application (Where Ordinal 0 is not working), you can use following steps to implement that.

  • Select the action
  • Change the ID Method to "JavaScript"
  • Add the following bold text Javascript in JavaScript:
    var x = Math.floor((Math.random() * 25) + 1); evalXPath("(//a[text()=\"view\"])["+x.valueOf()+"]");

Citrix scripting

You can use
"ctrx_logoff(CTRX_NORMAL_LOGOFF, CONTINUE_ON_ERROR, CTRX_LAST);"
before
"ctrx_disconnect_server("",CTRX_LAST);"

Even if your script breaks or fails in-between, it will logoff your citrix session. So, you no need to open and properly close the application manually.
